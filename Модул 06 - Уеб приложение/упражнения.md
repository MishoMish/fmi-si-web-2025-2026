## 11.01 — Fetch GET

Създайте HTML страница с бутон. При клик:
1. Изпратете GET заявка към `api.php`
2. PHP скриптът връща `{ "time": "текуща дата и час" }`
3. Покажете отговора в `<p>` елемент

## 11.04 — Пълно приложение: Бележник (Notes App)

Създайте мини уеб приложение за бележки:

### База данни
```sql
CREATE TABLE notes (
    id INT AUTO_INCREMENT PRIMARY KEY,
    title VARCHAR(100) NOT NULL,
    content TEXT,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);
```

### Frontend (HTML + JS)
- Форма за създаване на бележка (заглавие + съдържание)
- Списък с всички бележки (зареждат се при отваряне)
- Бутон „Изтрий" до всяка бележка

### Backend (PHP + PDO)
- `GET notes.php` — връща всички бележки като JSON
- `POST notes.php` — създава нова бележка
- `DELETE notes.php?id=X` — изтрива бележка

> Използвайте `fetch()` за всички заявки — **без** презареждане на страницата.

## 11.05 — Пълно приложение: Потребителска система

Създайте уеб приложение с потребителска автентикация:

### Структура
```
project/
├── homepage.html       ← главна страница
├── style.css
├── script.js
├── bootstrap.php       ← autoloader + session_start
├── session.php         ← GET (проверка), POST (логин), DELETE (логаут)
├── user.php            ← POST (регистрация)
└── classes/
    ├── .htaccess
    ├── Db.php
    ├── User.php
    └── Session.php
```

### Функционалност
1. **Регистрация** — форма → POST `user.php` → INSERT с хеширана парола
2. **Логин** — форма → POST `session.php` → проверка в БД → сесия
3. **Проверка на статус** — GET `session.php` → показване на бутон „Вход" или „Изход"
4. **Логаут** — DELETE `session.php` → `session_destroy()`
5. Навигационно меню с динамично показване на бутоните според статуса

### Бонус
- Страница „Профил" — достъпна само за логнати потребители
- Показване на имейла на логнатия потребител
- Валидация на входните данни (минимална дължина на парола, валиден имейл)
