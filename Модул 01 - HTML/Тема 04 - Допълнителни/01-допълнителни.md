# Допълнителни HTML теми

> Тази тема покрива допълнителен материал, който не е разгледан в предишните теми, но е важен за цялостното разбиране на HTML.

---

## 1. HTML Entities (Специални символи)

Някои символи в HTML имат специално значение (например `<` и `>` се използват за тагове). За да ги покажем като текст, използваме **HTML entities**.

### Основни entities

| Символ | Entity код | Описание |
|--------|-----------|----------|
| `<` | `&lt;` | По-малко (Less Than) |
| `>` | `&gt;` | По-голямо (Greater Than) |
| `&` | `&amp;` | Амперсанд |
| ` ` | `&nbsp;` | Неразделящ интервал (Non-Breaking Space) |
| `"` | `&quot;` | Кавички |
| `©` | `&copy;` | Авторско право |
| `€` | `&euro;` | Евро |

### Защо са нужни?

Ако напишете `<p>5 < 10</p>`, браузърът може да се обърка, защото `<` изглежда като начало на таг. Правилният начин е:

```html
<p>5 &lt; 10</p>
<p>10 &gt; 5</p>
<p>Фирма &amp; Ко.</p>
```

### `&nbsp;` — неразделящ интервал

HTML игнорира множество последователни интервали и ги свива до един. Ако искате да запазите допълнителен интервал, използвайте `&nbsp;`:

```html
<!-- Браузърът показва само един интервал -->
<p>Думи    с    много    интервали</p>

<!-- Всеки &nbsp; се запазва -->
<p>Думи&nbsp;&nbsp;&nbsp;&nbsp;с&nbsp;&nbsp;&nbsp;&nbsp;интервали</p>
```

`&nbsp;` също предотвратява пренасянето на нов ред между две думи — те винаги остават заедно.

## 2. Списък с дефиниции (`<dl>`, `<dt>`, `<dd>`)

Освен `<ul>` и `<ol>`, HTML предоставя и **списък с дефиниции** (Definition List), който се използва за термини и техните описания.

| Елемент | Описание |
|---------|----------|
| `<dl>` | Definition List — контейнер за списъка |
| `<dt>` | Definition Term — терминът |
| `<dd>` | Definition Description — описанието на термина |

### Пример

```html
<dl>
    <dt>HTML</dt>
    <dd>HyperText Markup Language — език за маркиране на уеб страници.</dd>

    <dt>CSS</dt>
    <dd>Cascading Style Sheets — език за стилизиране на уеб страници.</dd>

    <dt>JavaScript</dt>
    <dd>Програмен език за интерактивност в браузъра.</dd>
</dl>
```

### Кога да използваме `<dl>`?

- Речници и глосарии
- FAQ секции (въпрос → отговор)
- Метаданни (ключ → стойност)

```html
<dl>
    <dt>Автор</dt>
    <dd>Иван Петров</dd>

    <dt>Дата</dt>
    <dd>15 март 2026</dd>

    <dt>Категория</dt>
    <dd>Уеб разработка</dd>
</dl>
```

## 3. Допълнителни текстови елементи

### `<del>` — изтрит/премахнат текст

Показва текст, който е бил премахнат. Браузърът го визуализира с линия през него (~~задраскан~~):

```html
<p>Цената е <del>50 лв.</del> 35 лв.</p>
```

### `<ins>` — добавен/вмъкнат текст

Показва текст, който е бил добавен. Браузърът го визуализира с подчертаване:

```html
<p>Цената е <del>50 лв.</del> <ins>35 лв.</ins></p>
```

`<del>` и `<ins>` често се използват заедно, за да покажат промяна — например стара и нова цена.

### `<sub>` — долен индекс (Subscript)

Текстът се показва малко по-надолу от нормалната линия. Използва се за:

- Химични формули
- Математически индекси

```html
<p>Водата е H<sub>2</sub>O</p>
<p>CO<sub>2</sub> е въглероден диоксид</p>
```

### `<sup>` — горен индекс (Superscript)

Текстът се показва малко по-нагоре от нормалната линия. Използва се за:

- Степени и степенуване
- Бележки под линия

```html
<p>x<sup>2</sup> + y<sup>2</sup> = z<sup>2</sup></p>
<p>E = mc<sup>2</sup></p>
<p>3<sup>ти</sup> март</p>
```

## 4. Свързване на CSS и JavaScript

### Свързване на външен CSS файл (`<link>`)

За да свържем **външен CSS файл** с HTML документа, използваме тага `<link>` в `<head>`:

```html
<head>
    <meta charset="UTF-8">
    <title>Моята страница</title>
    <link rel="stylesheet" href="style.css">
</head>
```

| Атрибут | Описание |
|---------|----------|
| `rel="stylesheet"` | Указва, че файлът е стилова таблица |
| `href` | Пътят до CSS файла |

Могат да се свържат няколко файла:

```html
<link rel="stylesheet" href="reset.css">
<link rel="stylesheet" href="style.css">
<link rel="stylesheet" href="responsive.css">
```

### Тагът `<script>` — JavaScript

Тагът `<script>` се използва за добавяне на JavaScript код в HTML документа.

**Вграден (inline) JavaScript:**

```html
<script>
    alert("Здравей, свят!");
</script>
```

**Външен JavaScript файл:**

```html
<script src="app.js"></script>
```

| Атрибут | Описание |
|---------|----------|
| `src` | Пътят до външния JavaScript файл |

> **Важно:** Когато използвате атрибута `src`, тагът `<script>` не трябва да има съдържание вътре в него.

### Къде да поставим `<script>`?

Препоръчителната практика е `<script>` да се поставя в края на `<body>`, за да не блокира зареждането на страницата:

```html
<body>
    <h1>Съдържание</h1>
    <p>Текст...</p>

    <!-- Скриптовете отиват тук, в края на body -->
    <script src="app.js"></script>
</body>
```

## 5. Таг `<iframe>` — вграждане на друга страница

Тагът `<iframe>` вгражда друга HTML страница вътре в текущата:

```html
<iframe src="https://www.example.com" width="600" height="400" title="Примерен сайт"></iframe>
```

| Атрибут | Описание |
|---------|----------|
| `src` | URL на страницата за вграждане |
| `width` / `height` | Размери на рамката |
| `title` | Описание за достъпност |

### Практически приложения

- Вграждане на Google Maps
- Вграждане на YouTube видеа
- Показване на друг документ

```html
<!-- YouTube видео -->
<iframe width="560" height="315"
        src="https://www.youtube.com/embed/dQw4w9WgXcQ"
        title="YouTube video"
        allowfullscreen>
</iframe>

<!-- Google Maps -->
<iframe src="https://www.google.com/maps/embed?pb=..."
        width="600" height="450"
        title="Карта">
</iframe>
```

## 6. Мултимедия — `<video>` и `<audio>`

### `<video>` — вграждане на видео

HTML5 позволява директно вграждане на видео без плъгини:

```html
<video src="video.mp4" controls width="640" height="360">
    Вашият браузър не поддържа видео.
</video>
```

С множество формати за съвместимост:

```html
<video controls width="640">
    <source src="video.mp4" type="video/mp4">
    <source src="video.webm" type="video/webm">
    Вашият браузър не поддържа видео.
</video>
```

| Атрибут | Описание |
|---------|----------|
| `controls` | Показва контроли за управление (play, pause, volume и др.) |
| `autoplay` | Автоматично стартира видеото при зареждане |
| `loop` | Повтаря видеото безкрайно |
| `muted` | Заглушава звука |
| `poster` | Изображение, показвано преди пускане на видеото |
| `width` / `height` | Размери |

```html
<video controls autoplay muted loop poster="thumbnail.jpg" width="640">
    <source src="video.mp4" type="video/mp4">
</video>
```

> **Забележка:** Повечето браузъри блокират `autoplay` с включен звук. Ако искате автоматично пускане, добавете и `muted`.

### `<audio>` — вграждане на аудио

```html
<audio controls>
    <source src="song.mp3" type="audio/mpeg">
    <source src="song.ogg" type="audio/ogg">
    Вашият браузър не поддържа аудио.
</audio>
```

| Атрибут | Описание |
|---------|----------|
| `controls` | Показва контроли (play, pause, volume) |
| `autoplay` | Автоматично стартира при зареждане |
| `loop` | Повтаря безкрайно |
| `muted` | Заглушен по подразбиране |

## 7. Графики — `<canvas>` и `<svg>`

### `<canvas>` — растерна графика с JavaScript

Елементът `<canvas>` е правоъгълна област, в която можете да рисувате графики с JavaScript:

```html
<canvas id="myCanvas" width="400" height="300">
    Вашият браузър не поддържа canvas.
</canvas>

<script>
    const canvas = document.getElementById('myCanvas');
    const ctx = canvas.getContext('2d');

    // Рисуване на правоъгълник
    ctx.fillStyle = 'blue';
    ctx.fillRect(50, 50, 200, 100);

    // Рисуване на кръг
    ctx.beginPath();
    ctx.arc(300, 150, 50, 0, Math.PI * 2);
    ctx.fillStyle = 'red';
    ctx.fill();
</script>
```

**Кога да използваме `<canvas>`?**
- Игри в браузъра
- Графики и диаграми
- Анимации
- Обработка на изображения

### `<svg>` — векторна графика (Scalable Vector Graphics)

SVG е формат за **векторна графика**, която не губи качество при мащабиране:

```html
<svg width="200" height="200">
    <!-- Кръг -->
    <circle cx="100" cy="100" r="80" fill="green" stroke="black" stroke-width="2"/>

    <!-- Правоъгълник -->
    <rect x="10" y="10" width="80" height="60" fill="blue"/>

    <!-- Текст -->
    <text x="100" y="110" text-anchor="middle" fill="white" font-size="20">SVG</text>
</svg>
```

### `<canvas>` vs `<svg>`

| Характеристика | `<canvas>` | `<svg>` |
|----------------|-----------|---------|
| Тип графика | Растерна (пиксели) | Векторна (формули) |
| Мащабиране | Губи качество | Не губи качество |
| Рисуване | С JavaScript | С HTML/XML |
| Интерактивност | Ръчно с JS | Всеки елемент е DOM обект |
| Подходящ за | Игри, анимации | Лога, икони, диаграми |

## 8. Допълнителни елементи на формуляри

### `<optgroup>` — групиране на опции в `<select>`

Когато падащият списък има много опции, можем да ги групираме:

```html
<label for="car">Избери кола:</label>
<select id="car" name="car">
    <optgroup label="Немски">
        <option value="bmw">BMW</option>
        <option value="mercedes">Mercedes</option>
        <option value="audi">Audi</option>
    </optgroup>
    <optgroup label="Японски">
        <option value="toyota">Toyota</option>
        <option value="honda">Honda</option>
        <option value="nissan">Nissan</option>
    </optgroup>
</select>
```

Атрибутът `label` на `<optgroup>` показва заглавието на групата. Самото заглавие **не може да бъде избрано**.

### `<fieldset>` и `<legend>` — групиране на полета във форма

`<fieldset>` визуално и семантично групира свързани полета, а `<legend>` му дава заглавие:

```html
<form action="register.php" method="POST">
    <fieldset>
        <legend>Лична информация</legend>
        <label for="fname">Име:</label>
        <input type="text" id="fname" name="fname"><br>
        <label for="lname">Фамилия:</label>
        <input type="text" id="lname" name="lname">
    </fieldset>

    <fieldset>
        <legend>Акаунт</legend>
        <label for="uname">Потребителско име:</label>
        <input type="text" id="uname" name="uname"><br>
        <label for="pwd">Парола:</label>
        <input type="password" id="pwd" name="pwd">
    </fieldset>

    <button type="submit">Регистрация</button>
</form>
```

Браузърът рисува рамка около `<fieldset>` с текста от `<legend>` вградено в горния ръб.

### `<datalist>` — autocomplete за `<input>`

`<datalist>` дефинира списък от предложения за `<input>` поле. Потребителят може да избере от списъка или да напише свободен текст:

```html
<label for="browser">Браузър:</label>
<input list="browsers" id="browser" name="browser">

<datalist id="browsers">
    <option value="Chrome">
    <option value="Firefox">
    <option value="Safari">
    <option value="Edge">
    <option value="Opera">
</datalist>
```

Разликата между `<datalist>` и `<select>`:
- `<select>` — потребителят **задължително** избира от списъка
- `<datalist>` — потребителят **може** да избере от предложенията или да напише собствен текст

### Допълнителни типове `<input>`

Освен вече разгледаните в Тема 03 типове, ето и допълнителни:

| Тип | Описание | Пример |
|-----|----------|--------|
| `range` | Плъзгач (slider) | `<input type="range" min="0" max="100">` |
| `search` | Поле за търсене | `<input type="search" name="q">` |
| `tel` | Телефонен номер | `<input type="tel" name="phone">` |
| `url` | URL адрес | `<input type="url" name="website">` |
| `time` | Час | `<input type="time" name="meeting">` |
| `week` | Седмица | `<input type="week" name="week">` |
| `month` | Месец | `<input type="month" name="month">` |

#### Пример с `range` (плъзгач)

```html
<label for="volume">Сила на звука:</label>
<input type="range" id="volume" name="volume" min="0" max="100" value="50">
```

### Допълнителни атрибути на `<input>`

| Атрибут | Описание |
|---------|----------|
| `autofocus` | Автоматично фокусира полето при зареждане на страницата |
| `disabled` | Деактивира полето (не може да се редактира или изпрати) |
| `readonly` | Полето се вижда, но не може да се редактира |
| `autocomplete` | Включва/изключва автодовършване (`on` / `off`) |
| `multiple` | Позволява множествен избор (при `file` или `email`) |

```html
<!-- Полето автоматично получава фокус -->
<input type="text" name="search" autofocus placeholder="Търсене...">

<!-- Деактивирано поле -->
<input type="text" name="code" value="ABC123" disabled>

<!-- Поле само за четене -->
<input type="text" name="ref" value="REF-001" readonly>
```

## 9. `<tfoot>` — долна част на таблица

Освен `<thead>` (заглавна част) и `<tbody>` (тяло на таблицата), съществува и `<tfoot>` за долната част:

```html
<table border="1">
    <thead>
        <tr>
            <th>Продукт</th>
            <th>Количество</th>
            <th>Цена</th>
        </tr>
    </thead>
    <tbody>
        <tr>
            <td>Лаптоп</td>
            <td>2</td>
            <td>2400 лв.</td>
        </tr>
        <tr>
            <td>Мишка</td>
            <td>5</td>
            <td>150 лв.</td>
        </tr>
    </tbody>
    <tfoot>
        <tr>
            <td colspan="2"><strong>Общо:</strong></td>
            <td><strong>2550 лв.</strong></td>
        </tr>
    </tfoot>
</table>
```

| Елемент | Описание |
|---------|----------|
| `<thead>` | Групира заглавната част (хедър) на таблицата |
| `<tbody>` | Групира основната част (тяло) на таблицата |
| `<tfoot>` | Групира долната част (футър) на таблицата — обобщения, суми |

> **Забележка:** Можете да дефинирате `<tfoot>` преди `<tbody>` в кода, браузърът все пак ще го покаже в края на таблицата.

## 10. Булеви атрибути в HTML5

В HTML5 някои атрибути са **булеви** — те нямат стойност, а самото им присъствие означава `true`:

```html
<!-- Булев атрибут — само присъствието е достатъчно -->
<input type="text" name="name" required>
<input type="text" name="search" autofocus>
<video controls autoplay muted></video>
<details open></details>
<option selected></option>
```

### Валидни начини за записване на булев атрибут

```html
<!-- Всички тези записи са валидни в HTML5 -->
<input type="text" required>
<input type="text" required="required">
<input type="text" required="">
```

### Невалидни начини

```html
<!-- ❌ ГРЕШНО — required="true" НЕ е валидно -->
<input type="text" required="true">

<!-- ❌ ГРЕШНО — required="1" НЕ е валидно -->
<input type="text" required="1">
```

> **Важно:** Булев атрибут е активен, щом присъства. Стойността му може да бъде само празен низ или самото име на атрибута. `required="true"` **не е стандартен** HTML5 запис.

### Примери за булеви атрибути

| Атрибут | Елемент | Описание |
|---------|---------|----------|
| `required` | `<input>` | Полето е задължително |
| `autofocus` | `<input>` | Автоматичен фокус |
| `disabled` | `<input>`, `<button>` | Деактивиран елемент |
| `readonly` | `<input>`, `<textarea>` | Само за четене |
| `checked` | `<input type="checkbox/radio">` | Маркиран по подразбиране |
| `selected` | `<option>` | Избран по подразбиране |
| `multiple` | `<select>`, `<input type="file">` | Множествен избор |
| `controls` | `<video>`, `<audio>` | Показва контроли |
| `autoplay` | `<video>`, `<audio>` | Автоматично пускане |
| `muted` | `<video>`, `<audio>` | Заглушен звук |
| `loop` | `<video>`, `<audio>` | Повтаря безкрайно |
| `open` | `<details>` | Отворен по подразбиране |

## 11. HTML5 vs XHTML

### Какво е XHTML?

XHTML (eXtensible HyperText Markup Language) е по-стриктна версия на HTML, базирана на XML.

### Ключови разлики

| Характеристика | HTML5 | XHTML |
|----------------|-------|-------|
| Синтаксис | По-свободен | Стриктен (валиден XML) |
| Затварящи тагове | Незадължителни за някои тагове | Задължителни за **всички** тагове |
| Регистър на тагове | Не е case-sensitive | Всички тагове **трябва** да са lowercase |
| Самозатварящи се тагове | `<br>` е достатъчно | Изисква `<br />` |
| Атрибути | Могат да са без кавички | **Винаги** в кавички |
| DOCTYPE | `<!DOCTYPE html>` | По-дълга XML декларация |

### Примери

```html
<!-- ✅ Валиден HTML5 -->
<p>Текст
<br>
<img src="pic.jpg" alt=снимка>
<INPUT TYPE="TEXT">

<!-- ✅ Валиден XHTML (и HTML5) -->
<p>Текст</p>
<br />
<img src="pic.jpg" alt="снимка" />
<input type="text" />
```

> **Извод:** HTML5 има **по-свободен синтаксис** от XHTML. Въпреки това, добрата практика е да пишем чист и структуриран код, близък до XHTML стила.

## 12. Важни правила и концепции

### HTML5 не е case-sensitive

В HTML5 имената на таговете **не са** чувствителни към главни и малки букви:

```html
<!-- Всички тези записи са валидни -->
<P>Параграф</P>
<p>Параграф</p>
<P>Параграф</p>
```

> **Добра практика:** Винаги пишете тагове с **малки букви** — `<p>`, не `<P>`.

### `id` атрибутът е уникален

Атрибутът `id` трябва да е **уникален** в рамките на цялата страница. Не може два елемента да имат едно и също `id`:

```html
<!-- ✅ Правилно -->
<div id="header">Заглавие</div>
<div id="content">Съдържание</div>

<!-- ❌ Грешно — дублирани id -->
<div id="box">Кутия 1</div>
<div id="box">Кутия 2</div>
```

### `class` атрибутът може да е множествен

За разлика от `id`, един елемент може да има **множество класове**, разделени с интервал:

```html
<div class="card featured large">
    Елемент с три класа
</div>
```

Един клас може да се използва от **множество елементи**:

```html
<p class="highlight">Маркиран параграф 1</p>
<p class="highlight">Маркиран параграф 2</p>
<span class="highlight">Маркиран текст</span>
```

### Самозатварящи се (void) елементи

Някои HTML тагове **нямат** затварящ таг, защото не могат да съдържат други елементи. Те се наричат void (празни) елементи:

| Елемент | Описание |
|---------|----------|
| `<br>` | Нов ред |
| `<hr>` | Хоризонтална линия |
| `<img>` | Изображение |
| `<input>` | Поле за въвеждане |
| `<meta>` | Метаданни |
| `<link>` | Свързване на ресурси (CSS) |
| `<source>` | Източник за `<video>` / `<audio>` |

```html
<!-- ✅ Валидно в HTML5 -->
<br>
<hr>
<img src="pic.jpg" alt="Снимка">

<!-- ✅ Също валидно (XHTML стил) -->
<br />
<hr />
<img src="pic.jpg" alt="Снимка" />
```

### Кодировка (Charset) и нейното местоположение

Стандартната кодировка за HTML5 документи е **UTF-8**. Декларацията на кодировката се поставя в `<head>` секцията:

```html
<head>
    <meta charset="UTF-8">
    <!-- ... други елементи ... -->
</head>
```

> **Важно:** `<meta charset="UTF-8">` трябва да бъде **в `<head>` секцията** и е препоръчително да е на първия ред след отварящия `<head>` таг, за да може браузърът да знае кодировката преди да обработи останалото съдържание.

### Влагане на списъци

Списъците (`<ul>` и `<ol>`) **могат** да се влагат един в друг:

```html
<ul>
    <li>Frontend
        <ol>
            <li>HTML</li>
            <li>CSS</li>
            <li>JavaScript</li>
        </ol>
    </li>
    <li>Backend
        <ul>
            <li>PHP</li>
            <li>MySQL</li>
        </ul>
    </li>
</ul>
```

Вложеният списък трябва да бъде **вътре в `<li>` елемент** на родителския списък.

## 13. Обобщение — всички теми за подготовка

### Структура на HTML документа
- `<!DOCTYPE html>` — декларация за HTML5
- `<html>` — коренов елемент
- `<head>` — метаинформация, стилове, скриптове
- `<body>` — видимо съдържание
- `<title>` — заглавие на таба
- `<meta charset="UTF-8">` — кодировка (поставя се в `<head>`)

### Текстови елементи
- `<h1>` до `<h6>` — 6 нива заглавия
- `<p>` — параграф
- `<strong>` — важен (удебелен) текст
- `<em>` — наблегнат (наклонен) текст
- `<del>` — изтрит текст
- `<ins>` — добавен текст
- `<sub>` — долен индекс
- `<sup>` — горен индекс
- `<br>` — нов ред
- `<!-- коментар -->` — HTML коментар

### Линкове и изображения
- `<a href="...">` — хипервръзка; `target="_blank"` отваря в нов таб
- `<img src="..." alt="...">` — изображение

### Списъци
- `<ul>` / `<ol>` / `<li>` — неномериран / номериран списък
- `<dl>` / `<dt>` / `<dd>` — списък с дефиниции
- Списъците могат да се влагат

### Таблици
- `<table>`, `<tr>`, `<td>`, `<th>`
- `<thead>`, `<tbody>`, `<tfoot>`

### Формуляри
- `<form action="..." method="...">` — формуляр
- `<input>` — поле за въвеждане (`name` за достъп до стойността)
- `<label for="...">` — етикет
- `<textarea>` — многоредово поле
- `<button>` — бутон
- `<select>` / `<option>` / `<optgroup>` — падащо меню
- `<fieldset>` / `<legend>` — групиране на полета
- `<datalist>` — autocomplete предложения
- Input типове: `text`, `email`, `password`, `number`, `date`, `color`, `range`, `radio`, `checkbox`, `submit`
- Атрибути: `required`, `autofocus`, `placeholder`, `multiple`

### Семантични HTML5 елементи
- `<header>`, `<footer>`, `<nav>`, `<main>`, `<section>`, `<article>`, `<aside>`
- `<figure>` / `<figcaption>`

### Мултимедия и графики
- `<video>` — видео (`controls`, `autoplay`, `muted`)
- `<audio>` — аудио (`controls`, `autoplay`)
- `<canvas>` — растерна графика с JavaScript
- `<svg>` — векторна графика

### Свързване на ресурси
- `<link rel="stylesheet" href="style.css">` — външен CSS
- `<script src="app.js"></script>` — външен JavaScript
- `<iframe src="...">` — вграждане на друга страница

### HTML entities
- `&lt;` → `<`
- `&gt;` → `>`
- `&amp;` → `&`
- `&nbsp;` → неразделящ интервал

### Важни правила
- HTML5 **не е** case-sensitive (но пишете тагове с малки букви)
- `id` е **уникален** на страницата
- `class` може да е **множествен** (с интервал)
- Булевите атрибути се записват само с името: `required`, не `required="true"`
- HTML5 има **по-свободен синтаксис** от XHTML
- Семантичните тагове подобряват **достъпността и SEO**
- Стандартна кодировка: **UTF-8**
