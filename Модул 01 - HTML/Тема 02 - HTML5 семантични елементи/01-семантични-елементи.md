# HTML5 семантични елементи

## Какво е HTML5?

HTML5 е най-новата версия на HTML. Основната разлика спрямо старите версии е въвеждането на **семантични елементи**, които дават по-ясна структура и значение на съдържанието.

## Какво означава „семантичен"?

Семантичните елементи **описват смисъла** на съдържанието, което се намира вътре в тях. Те помагат на:

- **Търсачки** (Google) — по-добро индексиране
- **Браузъри** — по-добро рендериране
- **Помощни технологии** (екранни четци) — достъпност за хора с увреждания
- **Разработчици** — по-четим и поддържаем код

## Основни семантични елементи

```
┌─────────────────────────────────────┐
│               <header>              │
├─────────────────────────────────────┤
│                <nav>                │
├──────────────────────┬──────────────┤
│                      │              │
│       <main>         │   <aside>    │
│   ┌──────────────┐   │              │
│   │  <section>   │   │              │
│   │ ┌──────────┐ │   │              │
│   │ │<article> │ │   │              │
│   │ └──────────┘ │   │              │
│   └──────────────┘   │              │
│                      │              │
├──────────────────────┴──────────────┤
│               <footer>              │
└─────────────────────────────────────┘
```

| Елемент | Описание |
|---------|----------|
| `<header>` | Заглавна част на страница или секция |
| `<nav>` | Навигационно меню |
| `<main>` | Основното съдържание (едно на страница) |
| `<section>` | Тематичен блок от съдържание |
| `<article>` | Самостоятелно съдържание — статия, публикация |
| `<aside>` | Странично съдържание (допълнение) |
| `<footer>` | Долна част на страница или секция |

## `<section>` срещу `<div>`

| Елемент | Семантично значение | Типична употреба |
|---------|---------------------|------------------|
| `<section>` | ✅ Да | Тематични блокове |
| `<div>` | ❌ Не | Оформление и стилизиране |

`<div>` е подходящ, когато съдържанието няма ясен смислов блок, но трябва да се групира за стилизиране.

### Пример

**Със `<section>` (семантично):**

```html
<section>
    <h2>Новини</h2>
    <p>Последните публикувани статии и събития.</p>
</section>
```

**С `<div>` (без семантика):**

```html
<div>
    <h2>Новини</h2>
    <p>Последните публикувани статии и събития.</p>
</div>
```

Браузърът визуално ги показва еднакво, но `<section>` носи смисъл за машините.

## Пълен пример — структура на страница

```html
<!DOCTYPE html>
<html lang="bg">
<head>
    <meta charset="UTF-8">
    <title>Моят блог</title>
</head>
<body>
    <header>
        <h1>Моят блог</h1>
        <img src="logo.png" alt="Лого">
    </header>

    <nav>
        <ul>
            <li><a href="index.html">Начало</a></li>
            <li><a href="articles.html">Статии</a></li>
            <li><a href="contact.html">Контакти</a></li>
        </ul>
    </nav>

    <main>
        <section>
            <h2>Последни публикации</h2>

            <article>
                <h3>Въведение в HTML5</h3>
                <p>HTML5 добавя нови семантични елементи...</p>
                <footer>
                    <p>Публикувано на 15.03.2025</p>
                </footer>
            </article>

            <article>
                <h3>CSS Flexbox</h3>
                <p>Flexbox е мощен инструмент за подредба...</p>
                <footer>
                    <p>Публикувано на 22.03.2025</p>
                </footer>
            </article>
        </section>
    </main>

    <aside>
        <h2>За автора</h2>
        <p>Студент по Софтуерно инженерство, ФМИ.</p>
    </aside>

    <footer>
        <p>&copy; 2025 Моят блог. Всички права запазени.</p>
    </footer>
</body>
</html>
```

## Кога кой елемент да използваме?

- **`<article>`** — съдържание, което има смисъл само по себе си (статия, коментар, пост)
- **`<section>`** — тематичен блок, който е част от по-голямо цяло
- **`<div>`** — когато няма семантично значение, просто контейнер за CSS
- **`<header>` / `<footer>`** — могат да се използват вътре в `<article>` или `<section>`, не само на ниво страница

## `<figure>` и `<figcaption>`

Елементът `<figure>` обгръща самостоятелно съдържание (изображение, диаграма, код), а `<figcaption>` добавя надпис към него:

```html
<figure>
    <img src="chart.png" alt="Диаграма на продажбите за 2025">
    <figcaption>Фиг. 1 — Продажби по месеци за 2025 г.</figcaption>
</figure>
```

Може да се използва и за код:

```html
<figure>
    <pre><code>&lt;h1&gt;Здравей, свят!&lt;/h1&gt;</code></pre>
    <figcaption>Пример за HTML заглавие</figcaption>
</figure>
```

> **Важно:** `<figure>` не е само за снимки — подходящ е за всяко съдържание, което може да бъде преместено от основния текст, без да се загуби смисъл.

## Множество `<header>` и `<footer>` елементи

Честа грешка е да се мисли, че `<header>` и `<footer>` могат да се използват само веднъж. Всъщност всяка секция или статия може да има собствен `<header>` и `<footer>`:

```html
<article>
    <header>
        <h2>Заглавие на статията</h2>
        <p>Автор: Иван Петров | <time datetime="2025-11-20">20 ноември 2025</time></p>
    </header>

    <p>Съдържание на статията...</p>

    <footer>
        <p>Тагове: HTML, семантика, уеб</p>
    </footer>
</article>
```

## Елементът `<time>`

Семантичен елемент за дати и часове — машинно четим:

```html
<p>Публикувано на <time datetime="2025-03-15">15 март 2025</time></p>
<p>Концертът започва в <time datetime="20:00">20:00 ч.</time></p>
<p>Срокът е <time datetime="2025-12-31T23:59">31 декември 2025, 23:59</time></p>
```

Атрибутът `datetime` позволява на търсачките и браузърите да разпознаят точната дата, дори текстът да е изписан по различен начин.

## `<details>` и `<summary>` — сгъваемо съдържание

HTML5 позволява създаване на разгъващи се секции без JavaScript:

```html
<details>
    <summary>Кликни за повече информация</summary>
    <p>Тук се крие допълнително съдържание, което се показва при натискане.</p>
</details>

<details open>
    <summary>Тази секция е отворена по подразбиране</summary>
    <p>Атрибутът <code>open</code> прави секцията видима от началото.</p>
</details>
```

## `<mark>` — маркиран текст

Семантичен елемент за подчертаване на релевантен текст:

```html
<p>Резултати от търсенето: Езикът <mark>HTML</mark> е основата на уеб разработката.</p>
```

## `<address>` — контактна информация

Семантичен елемент за контактна информация на автора:

```html
<footer>
    <address>
        Написано от <a href="mailto:ivan@example.com">Иван Петров</a>.<br>
        Адрес: бул. Джеймс Баучер 5, София
    </address>
</footer>
```

> `<address>` **не** е за произволни адреси — само за контактна информация на автора на документа или статията.

## Достъпност (Accessibility) и семантика

Семантичните елементи автоматично създават **ARIA landmarks** (ориентири за навигация) за екранните четци:

| HTML елемент | ARIA роля | Значение за екранен четец |
|-------------|-----------|---------------------------|
| `<header>` | `banner` | Заглавна част на страницата |
| `<nav>` | `navigation` | Навигационно меню |
| `<main>` | `main` | Основно съдържание |
| `<aside>` | `complementary` | Допълнително съдържание |
| `<footer>` | `contentinfo` | Информация в края |
| `<section>` | `region` | Тематичен регион |
| `<article>` | `article` | Самостоятелно съдържание |

Това означава, че потребител с екранен четец може да "скочи" директно до навигацията, основното съдържание или футъра, без да слуша цялата страница.

## Често срещани грешки

### 1. Използване на `<section>` вместо `<div>` за стилизиране

```html
<!-- ❌ Грешно -->
<section class="wrapper">
    <div class="left-column">...</div>
    <div class="right-column">...</div>
</section>

<!-- ✅ Правилно -->
<div class="wrapper">
    <div class="left-column">...</div>
    <div class="right-column">...</div>
</div>
```

### 2. Липса на заглавие в `<section>`

Всяка `<section>` трябва да има заглавие (`<h2>`, `<h3>`, и т.н.):

```html
<!-- ❌ Грешно -->
<section>
    <p>Просто текст без заглавие...</p>
</section>

<!-- ✅ Правилно -->
<section>
    <h2>Заглавие на секцията</h2>
    <p>Текст на секцията...</p>
</section>
```

### 3. Множество `<main>` елементи

На страницата трябва да има **точно един** `<main>` елемент, и той **не** трябва да бъде вложен в `<article>`, `<aside>`, `<header>`, `<footer>` или `<nav>`.
