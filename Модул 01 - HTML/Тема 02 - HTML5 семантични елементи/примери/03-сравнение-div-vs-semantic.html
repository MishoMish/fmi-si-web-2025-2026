div soup vs Семантичен HTML

Идеята: И двата подхода произвеждат визуално идентичен резултат. Разликата е в значението за машините (търсачки, екранни четци) и четимостта за разработчиците.

Пример 1 — Навигация

Без семантика — код Семантичен HTML5 — код 
<div class="navigation">
  <div class="nav-list">
    <div class="nav-item">
      <a href="/">Начало</a>
    </div>
    <div class="nav-item">
      <a href="/about">За нас</a>
    </div>
    <div class="nav-item">
      <a href="/contact">Контакти</a>
    </div>
  </div>
</div>
 
<nav>
  <ul>
    <li>
      <a href="/">Начало</a>
    </li>
    <li>
      <a href="/about">За нас</a>
    </li>
    <li>
      <a href="/contact">Контакти</a>
    </li>
  </ul>
</nav>

Заключение: Семантичният HTML е по-кратък, по-четим и носи смисъл. Търсачките и помощните технологии разчитат на него, за да разберат структурата на страницата.
